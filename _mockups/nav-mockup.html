PLANNED FILE STRUCTURE
/components/
  header.html         ← markup only (injected by loader)
  footer.html         ← markup only (injected by loader)

/assets/css/
  lc-base.css         ← CSS reset + Google Fonts + CSS vars
  lc-header.css       ← header + mega-menu + mobile drawer
  lc-footer.css       ← footer styles
  lc-utils.css        ← shared utilities (grid, spacing, etc.)

/assets/js/
  lc-components.js    ← fetches + injects header.html & footer.html
  lc-nav.js           ← mega-menu hover, sticky, mobile accordion

template.html         ← base page skeleton (loads all of the above)
Injection strategy: lc-components.js uses fetch() to load /components/header.html and /components/footer.html and inserts them into #lc-header-mount / #lc-footer-mount placeholder divs on each page.
Works on GitHub Pages (no server-side includes needed).
